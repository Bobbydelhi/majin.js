# Contributing to Majin.js

First off, thank you for considering contributing to Majin.js! This project is being built from absolute zero, and community input is the core of its philosophy.

## The Goal
This is a university final project (TFG) aimed at proving that open-source collaboration can build better, lighter, and more innovative tools than corporate-backed monolithic frameworks. 

## How can I contribute?

### 1. Join the Discussion
We are starting from zero. If you have ideas for the architecture, the reactivity engine, or the custom syntax, open an Issue with the tag `discussion` or `architecture`.

### 2. Code Contributions
If you want to get your hands dirty:
1. Fork the repository.
2. Clone your fork locally.
3. Create a new branch for your feature or fix (`git checkout -b feature/your-feature-name`).
4. Write clean, documented Vanilla JavaScript. 
5. Commit your changes with descriptive messages.
6. Push to your fork and submit a Pull Request against the `main` branch.

### Core Philosophy
* **Zero Dependencies:** Do not add npm packages unless strictly necessary for tooling (like bundlers or testing). The core must remain pure JS.
* **Keep it Light:** Every byte counts.
* **Innovation over Convention:** Don't just copy Vue or React. Let's find better ways to do things.

We review every Pull Request and discuss it openly. Your code will literally be part of this thesis.
